# Multi-File AI Analysis Feature Design

## Overview
The multi-file AI analysis feature will allow users to select multiple processed documents and generate a comprehensive AI analysis that considers all selected files as a cohesive dataset.

## User Experience

### Frontend Changes
1. **New Button**: "AI Analysis" button appears when 2+ completed files are selected
2. **Analysis Modal**: Similar to AI Summary modal but with additional options:
   - Analysis Type: Comparative, Synthesis, Trends
   - Analysis Depth: Quick, Standard, Comprehensive
   - Optional custom prompt for specific analysis needs

### Workflow
1. User selects multiple completed files (2-20 files)
2. Clicks "AI Analysis" button
3. Selects analysis parameters in modal
4. System combines files with context markers
5. AI generates analysis across all documents
6. Result saved as new file with "_ANALYSIS" suffix

## Technical Implementation

### API Endpoint
```
POST /documents/multi-ai-analysis
{
  "document_ids": ["file1.md", "file2.md", ...],
  "analysis_type": "comparative|synthesis|trends",
  "analysis_depth": "quick|standard|comprehensive",
  "custom_prompt": "optional custom analysis instructions",
  "output_filename": "custom_analysis_name.md"
}
```

### Backend Processing
1. **Validation**:
   - Verify user owns all documents
   - Check file count limits (max 20)
   - Validate total content size (max 100KB combined)

2. **Document Preparation**:
   - Retrieve all documents from S3
   - Add document markers for context
   - Structure content for AI processing

3. **AI Processing**:
   - Use appropriate Bedrock model based on content size
   - Apply analysis-specific prompts
   - Generate structured analysis output

4. **Output Format**:
   ```markdown
   # Multi-File AI Analysis
   
   **Analysis Type**: Comparative Analysis
   **Documents Analyzed**: 5
   **Generated**: 2025-07-24 10:30:45
   **Model**: Claude 3 Haiku
   
   ## Executive Summary
   [Brief overview of key findings]
   
   ## Document Overview
   - Document 1: filename1.md (2.5KB)
   - Document 2: filename2.md (1.8KB)
   ...
   
   ## Analysis Results
   [Detailed analysis based on type selected]
   
   ## Key Insights
   [Bullet points of main findings]
   
   ---
   *Generated by Redact AI Analysis*
   ```

### Analysis Types

#### Comparative Analysis
- Identifies similarities and differences
- Highlights common themes
- Points out contradictions or conflicts
- Useful for policy comparison, contract review

#### Synthesis Analysis
- Combines information into unified summary
- Identifies key themes across documents
- Creates consolidated view
- Useful for research, meeting notes

#### Trends Analysis
- Identifies patterns over time
- Highlights changes and evolution
- Statistical insights where applicable
- Useful for reports, metrics, logs

### Cost Considerations
- Larger token usage than single summaries
- Model selection based on total content:
  - < 20KB: Claude 3 Haiku
  - 20-50KB: Claude 3 Haiku with chunking
  - > 50KB: Deny or use Claude Instant

### Implementation Priority
1. Basic multi-file selection and combination
2. Simple synthesis analysis with Haiku
3. Analysis type selection
4. Custom prompts
5. Advanced analysis features

## Security & Limits
- Max 20 files per analysis
- Max 100KB total content
- Rate limit: 5 analyses per hour per user
- Admin override for limits via config
